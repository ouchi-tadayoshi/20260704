# 要件定義書：Node.js版 簡易タスク管理（To-Do）CLIアプリ

## 1. プロジェクト概要
本ドキュメントは、Node.jsを用いたコマンドライン（CLI）ベースのタスク管理アプリケーションの開発における要件を定義するものである。

## 2. 開発・技術要件
- **実行環境**: Node.js
- **開発ツール**: Visual Studio Code, npm
- **ルートパス**: `http://localhost:3000/`
- **ディレクトリ構成**: 成果物一式を `/20260704/` に格納
- **データ保存形式**: CSVファイル

## 3. 機能要件

### 3.1 タスク操作（CRUD機能）
- **追加**: タスク名、期限、優先度を入力して新規登録する。
- **一覧表示**: 登録されたタスクをリスト形式で表示する。
- **編集**: 既存のタスク情報を修正する。
- **完了切替**: タスクの「未完了/完了」ステータスを更新する。
- **削除**: 指定したタスクをデータから削除する。

### 3.2 表示・ソート機能
- **表示切り替え**: 「すべて」「未完了のみ」「完了済みのみ」のフィルタリング機能を備える。
- **並び替え（ソート）**:
  - 初期状態：追加（作成）した順。
  - 切り替え：優先度順、または期限が近い順。

### 3.3 ユーザーインターフェース
- **操作形式**: アプリ起動後の対話型（インタラクティブ）メニュー形式。
- **エラーハンドリング**: 不正入力（型違い、範囲外など）時にはエラーメッセージを表示し、正しい値が入力されるまで再入力を求める。

## 4. データ定義

| 項目名 | データ型 | 制約・備考 |
| :--- | :--- | :--- |
| **タスク名** | 文字列 | |
| **期限** | 文字列 | `YYYY-MM-DD` 形式、または「今日」「明日」のキーワード |
| **優先度** | 数値 | 1 〜 5 の5段階 |
| **ステータス** | 区分 | 「未完了」または「完了」 |

## 5. 処理フロー
1. **起動**: アプリケーションを実行し、メインメニューを表示する。
2. **入力**: ユーザーが番号で操作を選択する。
3. **実行**: 選択された機能（追加・編集等）に応じた入力を受け付け、CSVデータを更新する。
4. **反映**: 処理完了後、結果を表示して再度メインメニューに戻る。
